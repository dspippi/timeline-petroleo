Visao geral

O projeto e uma aplicacao Next.js + React + TypeScript chamada Timeline do Petroleo. Ela mostra uma timeline horizontal de eventos historicos/geopoliticos ligados ao petroleo e permite comparar esses eventos com o preco historico do Brent.

O usuario navega com arrastar para pan, usa Ctrl + scroll para zoom e pode clicar em um evento para abrir um card com detalhes.

Site publico

Timeline + filtros + grafico de preco sincronizado.

Area admin

Edicao local de eventos e categorias (arquivos em data/).

Stack

Principais bibliotecas e responsabilidades:

  • next (App Router): paginas, rotas API e renderizacao server/client.
  • react: UI e interacao.
  • tailwindcss: estilos utilitarios.
  • date-fns: datas (parse/format).
  • recharts: grafico do preco.
  • framer-motion: animacoes (ex.: card do evento).
  • xlsx: import/export de eventos via Excel.

Fluxo principal (Home)

A pagina inicial app/page.tsx le os eventos no servidor e entrega para um wrapper cliente: TimelineClientWrapper.

  • Server: le data/events.json e cria uma lista de eventos com Date.
  • Serialize: converte datas para string ISO para passar ao client.
  • Client: reconverte strings em Date, aplica filtros, monta escala e renderiza UI.
Por que assim? O Next.js nao deixa enviar Date diretamente para Client Components. Por isso, a pagina serializa e o wrapper reidrata as datas.

Modelos de dados

O tipo central e o evento:

  • OilEvent: id, title, start_date, end_date?, country, region, type, company?, wikipedia?, description.
  • EventType e string aberta: suporta categorias customizadas em data/categories.json.
  • TimelineScale: converte Date -> pixel e pixel -> Date.

Um evento com end_date vira uma barra (intervalo). Um evento sem end_date vira um marcador pontual.

UI da timeline

O componente components/Timeline/Timeline.tsx coordena zoom, scroll e o eixo de anos. A lista visual de eventos e renderizada por TimelineRows.

  • Zoom: Ctrl + scroll (wheel) e botoes (in/out/reset/fit).
  • Pan: arrastar com mouse (drag-to-pan) atualiza scrollLeft/scrollTop.
  • Eixo anual: cabecalho fixo sincronizado com o scroll horizontal da timeline.
  • Hover: mousemove converte posicao X em data via scale.toDate e envia para o contexto.

Organizacao por regioes/paises

components/Timeline/TimelineRows.tsx agrupa eventos por regiao e por pais, renderiza headers e linhas, e distribui eventos em "lanes" quando ha sobreposicao horizontal.

Detalhe de performance: so renderiza eventos dentro do range visivel (com buffer), evitando custo alto em timelines longas.

Marcadores e tooltips

components/Timeline/EventMarker.tsx decide entre barra (intervalo) e losango (ponto) e controla tooltip/hover. Ao clicar, abre o EventCard.

Grafico de preco (Brent)

O grafico components/OilChart/OilPriceChart.tsx usa recharts e compartilha o mesmo eixo temporal da timeline, via scale.toPixel. Ele sincroniza com o hover e com o scroll horizontal.

  • Dados: o client chama /api/oil-prices via hooks/useOilPrices.ts.
  • API: app/api/oil-prices/route.ts usa lib/oilPrice.ts e faz cache/revalidate.
  • Fallback: combina JSON historico local e (opcionalmente) busca dados recentes da EIA com NEXT_PUBLIC_EIA_API_KEY.
  • Hover eficiente: o "overlay" do hover renderiza separado para nao re-renderizar o Recharts a 60fps.

Contextos globais

O layout raiz app/layout.tsx registra providers globais para sincronizacao e preferencias.

  • TimelineSyncProvider (context/TimelineSyncContext.tsx): guarda pxPerDay (zoom) e hoveredDate (hover).
  • SettingsProvider (context/SettingsContext.tsx): guarda altura de linha, tamanho de marcador, labels, dark mode.
  • CategoriesProvider (context/CategoriesContext.tsx): guarda lista de categorias e resolve cor/label por tipo.
Detalhe legal: o TimelineSyncContext separa estado e setters em contextos diferentes para reduzir re-renders, especialmente no hover.

Categorias (tipos de evento)

As categorias vivem em data/categories.json e sao lidas no server por lib/categories.ts. No client, o CategoriesProvider entrega getColor(type) e getLabel(type).

Como EventType e uma string aberta, voce pode criar categorias novas sem mexer no TypeScript.

Admin (CRUD em arquivo)

O admin e uma ferramenta para manutencao dos dados. A leitura/escrita acontece em data/events.json via funcoes server-only em lib/adminEvents.ts.

  • Persistencia: listEvents(), addEvent(), updateEvent(), deleteEvent().
  • API: app/api/admin/events/route.ts (GET lista, POST cria; existem rotas por id no repo).
  • Auth em producao: middleware.ts protege /admin e /api/admin via cookie/token.
  • Dev mode: em NODE_ENV=development, o middleware libera o admin (ferramenta local).

Dados e scripts

  • data/events.json: lista de eventos (fonte da timeline).
  • data/categories.json: lista de categorias e cores.
  • data/events.xlsx: base em Excel (para importar/exportar).
  • data/oil-prices-owid-historical.json e data/oil-prices-fallback.json: precos historicos locais.
  • scripts/events-excel.mjs: excel:import e excel:export.

Observacoes

  • Documentacao possivelmente desatualizada: o README menciona data/events.md, mas o codigo usa data/events.json.
  • Escala temporal: lib/timelineScale.ts define toPixel/toDate e uma borda extra para nao cortar labels nas extremidades.
  • Interacao polida: pan por drag nao usa estado React (mexe direto no DOM), e hover e "throttled" por frame.

Como rodar

  • Dev: npm run dev
  • Admin local: npm run admin (abre http://localhost:3000/admin e roda o dev server)
  • Excel: npm run excel:export / npm run excel:import
